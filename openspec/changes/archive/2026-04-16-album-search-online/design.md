## Context

当前 AlbumShelf 应用已有完整的专辑管理功能，包括：
- CSV 文件作为专辑收藏的数据源
- 通过 `ncm-cli` 与网易云音乐 API 交互
- `NcmCliService.searchAlbum()` 方法已实现，可搜索专辑
- 同步流程：CSV → SQLite → MusicBrainz 补全

本次变更需要在此基础上新增搜索入口和单张专辑添加能力。

## Goals / Non-Goals

**Goals:**
- 提供独立的搜索弹窗，不干扰现有专辑列表浏览体验
- 复用现有 `NcmCliService.searchAlbum()` 方法
- 添加专辑后立即同步，用户能马上看到新专辑
- 搜索结果中标识已收藏专辑，避免重复添加

**Non-Goals:**
- 不实现分页搜索（首次使用默认 10 条结果）
- 不实现搜索历史记录
- 不支持批量添加多张专辑

## Decisions

### 1. 搜索结果展示布局

**决策**：使用列表布局（封面 + 文字信息 + 操作按钮）

**理由**：
- 列表布局信息密度高，单屏可展示更多结果
- 与现有专辑表格风格保持一致
- 封面作为辅助视觉元素，不需要特别突出

**备选方案**：网格布局（大封面为主）—— 适合浏览型场景���但搜索场景更注重快速定位

### 2. 重复检测机制

**决策**：在前端渲染时检查 `netease_original_id` 是否已存在于本地数据库

**理由**：
- 搜索结果包含 `originalId`（网易云原始 ID）
- 本地专辑表有 `netease_original_id` 字段
- 前端可在渲染前调用 IPC 获取已收藏 ID 列表进行比对

**实现**：新增 IPC `album:getCollectedNeteaseIds` 返回所有已收藏专辑的 netease_original_id 集合

### 3. 添加流程的原子性

**决策**：添加操作分两步 —— 写入 CSV → 触发增量同步

**理由**：
- 保持 CSV 作为数据源的一致性（与现有导入流程统一）
- 增量同步只处理新增的这一张专辑，避免全量同步开销

**实现**：
1. `appendAlbumToCsv()` 追加一条记录到 CSV
2. 调用 `SyncManager.syncSingle()` 同步单张专辑
3. 自动触发 MusicBrainz 补全

### 4. 弹窗入口位置

**决策**：在顶部工具栏添加「🔍 搜索专辑」按钮

**理由**：
- 工具栏是全局操作入口，与同步、筛选等功能并列
- 用户无需进入菜单即可快速访问

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| 搜索请求需要网络，可能超时 | 显示加载状态，超时后提示重试 |
| ncm-cli 未登录时搜索可能失败 | 捕获 `NcmLoginRequiredError`，引导用户登录 |
| CSV 写入失败（权限/磁盘满） | 写入前备份，失败时回滚并提示 |
| 快速连续搜索导致请求堆积 | 使用 debounce，或取消前一个未完成的请求 |
