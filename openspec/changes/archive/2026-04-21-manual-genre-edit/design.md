## Context

当前专辑风格标签完全依赖 MusicBrainz 自动补全写入。数据库层已有完善的 genre + album_genre 多对多表结构，AlbumService 提供了 `setAlbumGenres(albumId, genreNames[])` 方法和 `getAllGenres()` 方法。前端详情展开区已展示风格标签，工具栏已有风格筛选自动补全组件可供参考。需要在此基础上增加手动编辑能力。

## Goals / Non-Goals

**Goals:**
- 用户可在专辑详情展开区就地编辑风格标签（添加、删除）
- 只允许从已有风格库中选择标签，不支持创建新风格
- 编辑态提供自动补全输入框，交互方式与工具栏风格筛选一致
- 手动保存模式，有保存和取消按钮

**Non-Goals:**
- 不支持拖拽调整标签顺序（按字母序自动排列）
- 不支持创建全新风格（只能选择 genre 表中已有的风格）
- 不支持批量编辑多张专辑的风格
- 不修改数据库表结构

## Decisions

### 1. 编辑入口：就地编辑按钮

在详情展开区「风格」label 旁增加一个 ✏️ 编辑按钮。点击后风格区域切换为编辑态，再次保存或取消后恢复查看态。

**为什么不用弹窗**：就地编辑更轻量，上下文不丢失，用户可以同时看到专辑其他信息。

### 2. 复用已有 AlbumService.setAlbumGenres()

后端不需要新建 Service 或方法，直接通过新的 IPC handler `album:setGenres` 调用已有的 `setAlbumGenres()`。

### 3. 风格来源限制

自动补全列表来自 `album:filters` 返回的 genres（即 genre 表中所有已有风格）。输入框只用于筛选，回车不创建新标签，只能从下拉列表中点选。这避免了拼写错误和风格名称不一致的问题。

### 4. 编辑态状态管理

使用组件局部状态管理编辑态：
- `editingGenreAlbumId: number | null` — 当前正在编辑风格的专辑 ID
- `editingGenres: string[]` — 编辑中的风格列表（保存前的临时副本）
- `genreEditInput: string` — 编辑态输入框的文本

保存时调用 IPC，成功后更新本地 albums 数据并退出编辑态。取消时丢弃临时副本。

## Risks / Trade-offs

- **[风格库为空]** → 如果用户从未做过 MusicBrainz 补全，genre 表为空，自动补全无可选项。可在编辑区显示提示文字"暂无可用风格，请先补全专辑数据"。
- **[编辑冲突]** → 用户编辑风格时，如果同时在执行自动补全，可能覆盖手动编辑结果。风险较低，因为补全和手动编辑通常不同时发生。当前不做额外处理。
