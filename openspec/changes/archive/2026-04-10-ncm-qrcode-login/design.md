## Context

AlbumShelf 是一个基于 Electron 的桌面应用，用于管理用户的网易云音乐专辑收藏。当前应用通过 `ncm-cli` 命令行工具与网易云音乐交互，已实现专辑同步、曲目查询、播放控制等功能。

目前应用缺少登录功能，导致：
1. 无法获取用户收藏的专辑数据（需要登录后才能访问用户歌单/收藏）
2. 部分 API 调用可能受限

现有架构：
- 主进程通过 `NcmCliService` 类调用 `ncm-cli` 命令
- 已有窗口菜单栏（数据、视图、窗口菜单）
- 渲染进程通过 preload 暴露的 API 与主进程通信

## Goals / Non-Goals

**Goals:**
- 实现网易云音乐扫码登录，用户可通过手机 App 扫码完成登录
- 在窗口菜单栏显示登录状态（用户昵称或"未登录"）
- 应用启动时自动检查登录状态，未登录时提示用户
- 登录成功后自动触发专辑同步
- 支持退出登录功能

**Non-Goals:**
- 不支持手机号密码登录、邮箱登录等其他登录方式（简化实现）
- 不在应用内持久化 cookie/token（由 ncm-cli 管理）
- 不实现复杂的会话管理（刷新 token 等由 ncm-cli 处理）

## Decisions

### 1. 使用 ncm-cli 管理登录状态

**决定**：所有登录相关操作都通过 ncm-cli 命令完成，不在 AlbumShelf 内部实现登录逻辑。

**理由**：
- ncm-cli 已经封装了网易云音乐 API 的认证逻辑
- cookie/session 由 ncm-cli 持久化管理，AlbumShelf 无需关心
- 复用现有 `NcmCliService` 架构，保持代码一致性

**替代方案**：直接调用网易云 API
- 优点：更灵活，不依赖外部工具
- 缺点：需要自己处理 cookie 持久化、加密解密、API 变化等复杂问题

### 2. 二维码轮询机制

**决定**：在渲染进程中定时调用 IPC 检查扫码状态，而非主进程主动推送。

**理由**：
- 简化实现，避免复杂的事件推送机制
- 二维码有效期约 3 分钟，轮询间隔 2 秒足够
- 用户主动触发登录，轮询期间用户在等待，性能影响可接受

**轮询流程**：
```
渲染进程                    主进程                     ncm-cli
   │                         │                          │
   │── 请求生成二维码 ────────>│                          │
   │                         │── login qrcode ─────────>│
   │                         │<─ {qrcode, key} ─────────│
   │<── 返回二维码 URL ────────│                          │
   │                         │                          │
   │── [轮询] 检查扫码状态 ───>│                          │
   │                         │── login status ─────────>│
   │                         │<─ {status, user?} ───────│
   │<── 返回状态 ──────────────│                          │
   │                         │                          │
```

### 3. 登录状态存储

**决定**：登录状态在主进程中维护一个内存变量，渲染进程通过 IPC 查询。

**理由**：
- 简单直接，符合 Electron 架构
- 状态变化时主进程通过 `webContents.send` 通知渲染进程
- 菜单栏更新需要在主进程操作，状态放主进程便于更新菜单

### 4. 菜单栏动态更新

**决定**：登录状态变化时，重新构建整个菜单并调用 `Menu.setApplicationMenu()`。

**理由**：
- Electron Menu 不支持动态修改单个菜单项的 label
- 菜单项数量少，重建成本低
- 保持代码简洁

## Risks / Trade-offs

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| ncm-cli 登录命令不支持或格式变化 | 登录功能失效 | 先验证 ncm-cli 支持的命令格式，做好错误处理 |
| 二维码过期用户未察觉 | 用户扫码失败困惑 | 显示过期提示，提供"刷新二维码"按钮 |
| 轮询期间网络波动 | 轮询失败 | 单次失败不中断轮询，连续多次失败才提示 |
| ncm-cli cookie 过期 | 启动时误判为已登录 | 定期检查登录状态有效性，失效时提示重新登录 |
